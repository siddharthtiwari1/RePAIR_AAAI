CODE vs PAPER MISMATCHES (found by reading the implementation)
================================================================================
These are not reviewer comments, but a reviewer could find them.

A. D_ref SIZE
   Paper: "a shared D_ref of 200 refusal prompts"
   Code:  make_idk_samples() returns ONE sample.
   -> The paper states a number the code does not support.

B. STEERING VECTOR IS BUILT FROM THE WRONG TENSOR
   compute_steering_vector() reads out.hidden_states[...] = the RESIDUAL STREAM
   output of the whole transformer block (after attention + MLP + residual).
   collect_H_O_prime() hooks the MLP module = the MLP OUTPUT, before the
   residual add.
   The code then does o = o + r_sv, i.e. adds a residual-stream direction to an
   MLP output. Both are R^4096 so it runs, but they are different distributions.
   The paper's Eq. (4) says r_SV comes from MLP_l(x). The code does not.

C. RANK
   Code argparse default = 32. Author states 64 was used. Confirm.

D. MAIN TABLE vs ABLATIONS DO NOT RECONCILE
   Main table, STAMP-LR, personal data: R-RL 0.88 | Utility 8.17 | RTE 4.01
     rank r=64      : 0.85 | 6.10 | 4.01
     rank r=128     : 0.88 | 6.07 | 5.24
     retain 1.00    : 0.90 | 7.07 | 4.01
     retain 0.10    : 0.88 | 8.83 | 3.12
   Utility 8.17 appears in NO ablation row. R-RL and RTE point to DIFFERENT
   rows. No single configuration produces all three.

E. SINGLE-SAMPLE TABLE IS IDENTICAL TO THE MAIN TABLE
   Main (|D_f| = 500):        Acc_f 0.00 | Acc_r 70.13 | Util 6.55
   Single-sample (|D_f| = 1): Acc_f 0.00 | Acc_r 70.13 | Util 6.55
   Byte-identical for both STAMP and STAMP-LR. Forgetting 1 sample and 500
   cannot give identical retain accuracy and perplexity to two decimals.
   This undercuts the headline single-sample claim -- the paper's strongest
   differentiator.

F. "IDK Rate" is a column in Table 3 but is never defined in Metrics.

G. DATASET SPLIT CONTRADICTION
   Results: "each dataset is split equally into D_f and D_r" (50/50)
   Problem Formulation footnote: "D_r is at most 10% of D \ D_f"
   Both cannot be true.

H. \input{section/8_values} sits INSIDE 4_method.tex. The results table is
   therefore numbered Table 1 and floats into the Method section.

I. Dangling \label{sec:layer_analysis} on a \textbf line, not a heading.
